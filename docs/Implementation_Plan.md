- обновление state выполняется только после успешного завершения pipeline

---

### Слои данных

- `data/raw/variant_03/` — исходные данные API (JSON)
- `data/normalized/variant_03/` — очищенные табличные данные
- `data/mart/variant_03/` — агрегированная витрина

---

### Watermark

- используется поле: `date`
- определяется как максимальная дата в mart
- обновляется после успешного выполнения всех этапов

Пример:
watermark = 2026-04-14

---

### Business key

Одна строка витрины = один день по одному городу

Business key:
date + city_id

---

### Идемпотентность

- повторный запуск пайплайна не изменяет результат при одинаковых входных данных
- загрузка в PostgreSQL выполняется через `if_exists="replace"`
- дубликаты не накапливаются

---

### Режимы работы

#### Full

- выполняются все этапы заново
- все слои пересоздаются
- витрина полностью пересобирается
- таблица в БД заменяется

#### Incremental

- используется `state.json`
- сохраняется и используется watermark
- пайплайн можно безопасно запускать повторно
- результат не дублируется

---

### Результат

pipeline автоматизирован и управляется одной командой  
реализована логика state и watermark  
обеспечена идемпотентность  
поддерживаются режимы full и incremental  

---

### Итог

Собран полноценный ETL pipeline:

- единая точка входа
